                                                 CURRENT PATCH NOTES
=======================================================================================================================

**v7.0.0**

- **Addon Restrictions Compliance**
  - Updated the addon to comply with new Blizzard addon restrictions regarding combat log access and protected functions.
  - Removed all functionality that relied on `COMBAT_LOG_EVENT_UNFILTERED` to prevent `ADDON_ACTION_FORBIDDEN` errors.
  - Added combat checks to various modules to prevent restricted API calls during combat.
  - Ensured secure frames are handled correctly to avoid taint issues.

**v6.2.0**  

**NOTE**:  Because of some significant under-the-hood changes, all options **must** be reset when installing this version.
           You will be prompted to do so when you first log in.  Click Yes, and then you can re-configure things.

- **Performance & Efficiency**  
  - Added throttling for high-frequency events to reduce CPU load  
  - Centralized event handling into a single system for efficiency  
  - Reworked Alerts module to significantly reduce overhead, especially in combat  
  - Fixed unintended resource usage while in combat, dead, or a ghost  
  - General code formatting and cleanup  

- **New Features**  
  - Added all Mage portals to the Alerts module (localized spell names supported)  
  - Added missing food items: Authentic Undermine Clam Chowder, Hearty Authentic Undermine Clam Chowder  
  - Added Algari Repair Bot 11O to the repair bot Alert  
  - Added a slash command to fully reset settings and SavedVariables: /crb reset, /buff reset, /funki reset  
  - Added in-group requirement for single-target raid buffs (e.g., Source of Magic, Symbiotic Relationship)  
  - Added range checks for group-wide raid buffs, hiding the icon if no eligible targets are in range  
  - Added Recuperate, which displays out of combat if below 100% health and not in a rested area  

- **Options Menu Improvements**  
  - Introduced a new Customization tab for various addon options  
  - Renamed the Raid Announcer module and tab to “Alerts”  
  - Added an option to disable the Alerts module while in combat  
  - Moved Mythic+ options into a dedicated sub-tab under Customization  
  - Added an option to reverse the display order of Hunter pets  
  - Added an option to disable the Hunter pet specialization icon  
  - Added an option to swap the Hunter pet’s icon with its special ability icon  
  - Added an option to show the Hunter pet’s special ability icon, instead of the pet's icon, on mouseover 
  - Added an option to disable tooltips on icons  

- **System Updates**  
  - Restructured glow color handling for consistency and easier expansion  
  - Runeforge Reminder (Death Gate) now uses the Special glow color  
  - Reworked gating logic for a more modular and flexible design  
  - Renamed some tabs in the Options menu
  - Rebuilt the Trinkets module for improved reliability;  

- **Bug Fixes**  
  - Cosmetic items are now properly ignored by default on new installs  
  - Hearty Feast of the Midnight Masquerade now correctly triggers feast announcements  
  - Eating icon now properly detects all localized buff names and hides food icons while eating
  - Eating icon now clears when Well Fed or Hearty Well Fed is gained
  - Fixed Blessing of the Bronze not detecting all raid copies of the spell  
  - Fixed Shaman Shields entries on the Ignore List not functioning  
  - Fixed timers hanging at 0:01 instead of clearing at 0:00  
  - Fixed icons not refreshing properly when reaching the configured threshold  
  - Fixed weapon oils not showing with bows, guns, or wands equipped  
  - Fixed castable weapon enchants (e.g., Flametongue Weapon) not displaying properly
  - Fixed cosmetic items not being ignored on fresh installs  
  - Fixed Enable Glow checkbox desync on reload (appeared checked but inactive until re-toggled)  
  - Fixed an issue that caused So’Leah’s Secret Technique to not display reliably
    - This item does not yet create a macro to re-cast on the previous target (planned for a future update)
  - Fixed an issue that could cause an error if reloading the UI while in combat
  - Fixed an issue that could cause the buttons on the Mover frame to load improperly

- **UI & Quality of Life**  
  - Class Abilities on the Ignore tab now prioritize the player’s class first  
  - Added addon version number to the Info tab of the Options menu
